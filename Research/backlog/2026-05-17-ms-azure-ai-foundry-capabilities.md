# Microsoft Azure AI Foundry: full feature and capability survey

## Research Question

What is the complete set of features, functions, and capabilities offered by Microsoft Azure AI Foundry, and how do those capabilities support the full Artificial Intelligence (AI) development lifecycle — from model selection and fine-tuning through deployment, evaluation, and production governance — in a regulated enterprise?

## Scope

**In scope:**
- Azure AI Foundry model catalogue: available foundation models, model benchmarking, and selection tooling
- Model customisation: fine-tuning, continued pre-training, distillation
- Development tools: Prompt Flow, Azure AI Foundry SDK, agent building capabilities, playground environments
- Evaluation and testing: built-in evaluation metrics, safety evaluations, red-teaming tools
- Deployment and serving: managed endpoints, serverless API options, deployment configurations
- Responsible AI and safety tooling: content safety filters, groundedness detection, fairness tooling
- Observability and monitoring: tracing, logging, cost and usage dashboards
- Enterprise security and access controls: private endpoints, managed identity, role-based access control (RBAC), data residency
- Integration with Azure Machine Learning (ML), Azure OpenAI Service, and Microsoft Copilot products
- Licensing, pricing model, and resource management
- Product history: transition from Azure AI Studio to Azure AI Foundry, what changed

**Out of scope:**
- Azure Machine Learning (ML) capabilities beyond their Foundry integration surface
- Microsoft Copilot Studio (covered in a separate research item)
- On-premises or disconnected deployment scenarios unless directly supported by Foundry

**Constraints:** Azure AI Foundry is the rebrand and expansion of Azure AI Studio (rebranded late 2024); include transition context where relevant; focus on generally available features; note preview features explicitly.

## Context

Azure AI Foundry is Microsoft's unified platform for AI application development, covering model access, fine-tuning, evaluation, and production deployment. It is the developer-facing complement to Copilot Studio's low-code agent builder. Understanding its full capability set informs enterprise AI architecture decisions, the division of responsibilities between Azure AI Foundry and Copilot Studio, and vendor selection for regulated AI workloads.

## Approach

1. What model selection and management capabilities does Azure AI Foundry provide?
   1a. What models are in the catalogue and through what access mechanisms (Azure OpenAI, open-weight, third-party)?
   1b. What fine-tuning and model customisation options are available?
   1c. What benchmarking and model evaluation tools are provided?
2. What development tools exist for building AI applications?
   2a. What is Prompt Flow and what workflows does it support?
   2b. What agent building capabilities are available (Azure AI Agents Service)?
   2c. What Software Development Kits (SDKs) and APIs are provided?
3. What evaluation and testing capabilities are available for AI applications?
4. What deployment and serving options are supported?
5. What governance, safety, and responsible AI features are built in?
   5a. What content safety and moderation controls are available?
   5b. What audit, tracing, and compliance features apply to regulated industries?
6. What observability, monitoring, and cost management capabilities exist?
7. How does Azure AI Foundry integrate with Azure OpenAI Service, Azure ML, and Microsoft Copilot products?
8. What are the licensing model, resource provisioning model, and cost structure?

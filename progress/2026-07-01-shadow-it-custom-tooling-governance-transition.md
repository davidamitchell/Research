**Key findings summary:**
The benefit (5 categories) and risk (5 categories) taxonomy for shadow IT is stable across two independent systematic reviews, with "continuity lack" (single-person dependence, absent documentation) directly naming the tacit-knowledge/staff-departure mechanism the research question asked about. Neither primary review quantifies the lifecycle cost of a continuity failure in monetary or time terms; the closest available quantification is a companion repository item's fragmentation telemetry (individual task completion +33.7%, pull request review time +441% across 22,000 developers), which measures aggregate cost rather than a single event. The best cross-source-supported governance approach is a staged identify-evaluate-allocate sequence (not prohibition or awareness training alone, both empirically shown ineffective), corroborated by companion items on citizen-development governance and platform engineering. A companion item's golden path pattern is a complementary front-end mechanism (prevents new shadow IT) distinct from the back-end staged transition sequence (handles existing instances).

Two review passes were needed instead of one: the first review-fix pass addressed the reported violations but introduced two smaller new ones (a Sources-list gap for a newly cited source, and an unexpanded acronym) that a broader self-check before triggering review would have caught.
3. **What single change would prevent this next time?** When adding a *new* citation to a source during a review-fix pass (not originally in `## Sources`), add it to the Sources checklist in the same edit pass, not as an afterthought -- this is a mechanical rule, not a judgment call, and should be checked immediately after any Analysis/Findings edit that introduces a new external URL. Also: when large multi-paragraph `edit` replacements are planned, extract the exact existing block via `sed`/`view` first and diff it against the intended `old_str` before calling `edit`, rather than relying on memory of the file's exact wording.

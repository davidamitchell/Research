## Mini-Retro

1. **Did the process work?** Yes. Sequential unlocking (W-0076 → W-0077 → W-0078 → W-0079 → W-0080 → W-0081) held throughout. TDD (RED → GREEN → REFACTOR) was applied on every step. `make check` caught 3 lint/format issues before committing.

2. **What slowed down or went wrong?** The `_item()` test helper in `test_canonicalise_themes.py` used `textwrap.dedent` with f-string interpolation — when the substituted content had no leading whitespace, `textwrap.dedent` left the surrounding template lines indented, producing malformed YAML files. Caught by the first RED run; fixed by writing `f"---\n{fm}\n---\n\n{body}\n"` directly.

3. **What single change would prevent this next time?** The `textwrap.dedent` + f-string multiline template pattern is fragile for frontmatter helpers. The simpler `f"---\n{textwrap.dedent(fm).strip()}\n---\n\n{body}\n"` pattern should be the standard for all test file-writing helpers.

4. **Is this a pattern?** Yes — this is a new instance of the "test helper produces malformed fixtures" class. Not previously listed. Adding to the Known Recurring Failure Patterns table in `copilot-instructions.md` would help.

5. **Does any documentation need updating?** `docs/themes-vocabulary.md` is the new vocabulary source of truth. `Research/_template.md` and `research-prompt.md` updated. ADR-0018 written.

6. **Do the default instructions need updating?** The Known Recurring Patterns table in `.github/copilot-instructions.md` should note the `textwrap.dedent` + f-string multiline helper risk, but that's a minor addition. The more important update is already in place: `docs/themes-vocabulary.md` exists and the template references it.

## Summary

The issue asks for a survey of existing IT capability models that define multi-level technical capabilities (authentication, networking, API gateways, data storage, etc.), with the goals of:
1. Supporting implementation-agnostic solution design
2. Providing a standardised maturity assessment framework

The backlog item captures the full research question, scope, context, approach, and a comprehensive starting sources list covering:
- Traditional architecture frameworks: TOGAF (The Open Group Architecture Framework), SABSA (Sherwood Applied Business Security Architecture), DoDAF (Department of Defense Architecture Framework), NAF (NATO Architecture Framework), BIAN (Banking Industry Architecture Network), TM Forum (formerly TeleManagement Forum), NIST (National Institute of Standards and Technology) frameworks
- Maturity models: IT-CMF (IT Capability Maturity Framework), CMMI (Capability Maturity Model Integration)
- Modern/engineering-focused approaches: Wardley Mapping, Team Topologies, DORA (DevOps Research and Assessment), cloud Well-Architected Frameworks (AWS (Amazon Web Services), Azure, Google)
- CSDM (Common Service Data Model) integration analysis

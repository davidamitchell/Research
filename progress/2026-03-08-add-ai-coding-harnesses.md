## Research question captured

> What are the core architectural and philosophical principles behind the AI coding harnesses (agentic IDEs and agent runtimes) published by Anthropic, OpenAI, and peers? Specifically: where and how do the agents run, what do they have access to (filesystem, tools, APIs), and how do they handle memory, state, progress management, and context window management?

## Key dimensions scoped

1. Execution environment (local process, cloud sandbox, container)
2. Tool access patterns (filesystem, APIs, codebase)
3. Memory model (in-context, external store, scratchpad, episodic vs semantic)
4. Context window management (truncation, summarisation, compression)
5. Progress and state management (multi-step task tracking, retry, human handoff)
